Artefact

This folder contains documentation and reference material for the research implementation. It does not include source code. The actual implementation resides in a separate repository.
The goal of this artefact is to provide sufficient context to understand, reproduce, and cite the work and its outputs.

Structure

The directory is organized as follows:

artefact/
├── source_code_info/
├── data/
└── results/
source_code_info/

This folder contains metadata and references about the implementation rather than the code itself.

Typical contents include:

Link to the source code repository
Specific commit hash or release/tag used for the study
High-level system architecture description
List of dependencies and runtime environment details (e.g., language version, OS, hardware requirements)
Setup and cloning instructions
Known limitations or configuration notes
data/

This folder contains information about datasets used in the implementation, or references to external data sources.

It may include:

Dataset descriptions and schemas
Links to external datasets (if not stored locally)
Data preprocessing notes
Access instructions or credentials requirements (if applicable)
results/

This folder stores outputs generated by the implementation, such as:

Experiment results
Evaluation metrics
Logs or reports
Visualizations (charts, plots, figures)
Final model outputs or summaries
Maintenance
Keep all references in source_code_info/ updated whenever the codebase changes in a way that affects reproducibility.
Do not commit large or sensitive datasets directly (use .gitignore and external storage when necessary).
Ensure commit messages are clear and descriptive to maintain traceability.
